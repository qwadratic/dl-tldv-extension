  <action>
1. Create `src/types.ts` with:
   ```typescript
   // Shared types for dl-tldv extension
   export interface DownloadMessage {
     type: "START_DOWNLOAD";
     meetingId: string;
   }

   export interface ProgressMessage {
     type: "DOWNLOAD_PROGRESS";
     current: number;
     total: number;
   }

   export interface ErrorMessage {
     type: "DOWNLOAD_ERROR";
     error: string;
   }

   export type ExtensionMessage = DownloadMessage | ProgressMessage | ErrorMessage;
   ```

2. Create `src/manifest.json`:
   ```json
   {
     "manifest_version": 3,
     "name": "tldv Meeting Downloader",
     "version": "0.1.0",
     "description": "Download tldv.io meeting recordings as MP4 files",
     "permissions": ["activeTab"],
     "host_permissions": [
       "https://*.tldv.io/*",
       "https://gw.tldv.io/*",
       "https://gaia.tldv.io/*"
     ],
     "background": {
       "service_worker": "background.js"
     },
     "content_scripts": [
       {
         "matches": ["https://*.tldv.io/*"],
         "js": ["content.js"],
         "run_at": "document_idle"
       }
     ],
     "icons": {},
     "browser_specific_settings": {
       "gecko": {
         "id": "dl-tldv@extension.local",
         "strict_min_version": "109.0"
       }
     }
   }
   ```

   Key points:
   - `host_permissions` includes tldv.io, gw.tldv.io (API), and gaia.tldv.io (playlist) for Phase 2
   - `content_scripts.matches` targets all tldv.io pages; the content script itself will detect meeting vs non-meeting
   - `browser_specific_settings.gecko` provides Firefox compatibility with gecko ID and min version 109.0 (first Firefox MV3 release)
   - `run_at: "document_idle"` ensures DOM is ready before injection

3. Create `src/background.ts`:
   ```typescript
   import browser from "webextension-polyfill";

   // Service worker entry point
   // Phase 2 will add download pipeline logic here
   browser.runtime.onInstalled.addListener(() => {
     console.log("[dl-tldv] Extension installed");
   });

   browser.runtime.onMessage.addListener((message, sender) => {
     console.log("[dl-tldv] Message received:", message);
     // Phase 2: handle START_DOWNLOAD messages
     return undefined;
   });
   ```

4. Create `src/content.ts`:

   ```typescript
   // Content script entry point
   // Phase 1 Plan 02 will add button injection logic here

   ```

5. Run `npm run build` and verify `dist/` contains: `background.js`, `content.js`, `manifest.json`.
